# Settings Page Audit

## Skill Name

Settings Page Audit

## Description

Audit settings, preferences, admin, and account pages for findability, grouping, safety, permissions, forms, and change confirmation.

## When To Use

- Settings pages have grown organically and are difficult to scan.
- Users cannot find account, billing, permissions, notification, or integration controls.
- A settings flow includes dangerous actions, role-based access, or complex forms.
- A team wants to professionalize a SaaS or admin configuration surface.

## Inputs Required

- Settings routes, screenshots, sitemap, component code, or product requirements.
- User roles, permissions, and ownership boundaries.
- List of settings, destructive actions, and confirmation requirements.
- Known support tickets, search terms, or usability concerns.

## Step-by-Step Workflow

1. Inventory settings by domain, role, frequency, and risk.
2. Review grouping, labels, search, navigation, save behavior, defaults, and unsaved-change handling.
3. Check destructive actions, permissions, confirmations, auditability, and error recovery.
4. Evaluate form accessibility, validation, loading states, and success feedback.
5. Recommend IA, copy, interaction, and engineering changes in priority order.

## Output Format

- Settings audit report with grouped findings and severity.
- Recommended settings IA and action safety improvements.
- Checklist for forms, permissions, destructive actions, and responsive behavior.

## Success Criteria

- Users can find and change settings without guessing or losing work.
- Dangerous actions are protected without blocking normal workflows.
- Permissions and ownership are clear.
- Save, success, failure, and unsaved-change states are predictable.

## Common Mistakes

- Grouping settings by implementation owner instead of user task.
- Using one global save model when sections have different risk levels.
- Hiding destructive actions without clear recovery or confirmation.
- Skipping role and permission edge cases.

## Example Usage

Use this skill to audit organization settings covering team members, roles, billing, integrations, notifications, and account deletion.
