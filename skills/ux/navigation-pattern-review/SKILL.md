# Navigation Pattern Review

## Skill Name

Navigation Pattern Review

## Description

Review navigation patterns such as sidebars, tabs, breadcrumbs, menus, command palettes, and mobile nav for clarity and wayfinding.

## When To Use

- Users struggle to move between product areas or understand where they are.
- A product is adding new sections to an existing navigation model.
- Navigation differs across desktop, tablet, and mobile views.
- A team needs to choose between tabs, sidebars, breadcrumbs, menus, or search.

## Inputs Required

- Navigation screenshots, routes, sitemap, component code, or prototype.
- Primary user tasks, content hierarchy, and permissions model.
- Supported devices, breakpoints, and interaction methods.
- Analytics, search logs, support tickets, or known findability issues.

## Step-by-Step Workflow

1. Map navigation levels, entry points, current-location indicators, and escape paths.
2. Evaluate labels, grouping, order, hierarchy, responsive behavior, and keyboard accessibility.
3. Check whether the pattern matches task frequency, information depth, and user mental model.
4. Identify hidden destinations, duplicated labels, dead ends, and permission-related confusion.
5. Recommend navigation structure and interaction changes with migration considerations.

## Output Format

- Navigation review with findings, severity, and recommended pattern changes.
- Proposed navigation model or IA adjustments when needed.
- Responsive and accessibility checklist for implementation.

## Success Criteria

- Users can predict where destinations live and recover when they navigate incorrectly.
- Current location and hierarchy are visible and accessible.
- Navigation scales as content grows without becoming cluttered.
- Recommendations account for permissions, routing, and responsive constraints.

## Common Mistakes

- Choosing a pattern because it is visually trendy rather than task-appropriate.
- Hiding high-frequency actions in overflow menus.
- Ignoring keyboard and screen reader behavior for custom menus.
- Renaming navigation items without considering user terminology.

## Example Usage

Use this skill to review a SaaS app moving from top navigation to a sidebar with nested settings, billing, projects, and account sections.
