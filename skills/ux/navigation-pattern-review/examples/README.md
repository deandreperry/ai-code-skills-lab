# Navigation Pattern Review Examples

## Sample Prompt

```text
Use the Navigation Pattern Review skill.
Inputs:
- Navigation screenshots, routes, sitemap, component code, or prototype.
- Primary user tasks, content hierarchy, and permissions model.
- Supported devices, breakpoints, and interaction methods.
```

## Example Scenario

Use this skill to review a SaaS app moving from top navigation to a sidebar with nested settings, billing, projects, and account sections.

## Expected Output

- Navigation review with findings, severity, and recommended pattern changes.
- Proposed navigation model or IA adjustments when needed.
- Responsive and accessibility checklist for implementation.
