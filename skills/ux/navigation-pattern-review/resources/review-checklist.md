# Navigation Pattern Review Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm navigation screenshots, routes, sitemap, component code, or prototype.
- [ ] Confirm primary user tasks, content hierarchy, and permissions model.
- [ ] Confirm supported devices, breakpoints, and interaction methods.
- [ ] Confirm analytics, search logs, support tickets, or known findability issues.

## Execution

- [ ] Map navigation levels, entry points, current-location indicators, and escape paths.
- [ ] Evaluate labels, grouping, order, hierarchy, responsive behavior, and keyboard accessibility.
- [ ] Check whether the pattern matches task frequency, information depth, and user mental model.
- [ ] Identify hidden destinations, duplicated labels, dead ends, and permission-related confusion.
- [ ] Recommend navigation structure and interaction changes with migration considerations.

## Completion

- [ ] Users can predict where destinations live and recover when they navigate incorrectly.
- [ ] Current location and hierarchy are visible and accessible.
- [ ] Navigation scales as content grows without becoming cluttered.
- [ ] Recommendations account for permissions, routing, and responsive constraints.
