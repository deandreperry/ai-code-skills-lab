# Onboarding Flow Review

## Skill Name

Onboarding Flow Review

## Description

Review product onboarding flows for activation, clarity, sequencing, friction, trust, accessibility, and first-value delivery.

## When To Use

- A signup, setup, or first-run experience needs UX review.
- Activation metrics show users dropping before reaching value.
- A product is adding guided setup, templates, tours, or checklists.
- Designers and engineers need a shared plan for reducing onboarding friction.

## Inputs Required

- Onboarding screens, routes, prototype, analytics, or screenshots.
- Activation goal, primary user segment, and success metric.
- Required setup steps, optional steps, and data dependencies.
- Known drop-off points, support feedback, or usability findings.

## Step-by-Step Workflow

1. Map the onboarding journey from entry point to first meaningful value.
2. Separate required decisions from optional personalization and education.
3. Review copy, progress indication, defaults, permissions, error recovery, and escape paths.
4. Identify friction, premature complexity, trust gaps, accessibility issues, and missing states.
5. Recommend sequence, content, interaction, and instrumentation improvements.

## Output Format

- Onboarding flow assessment with friction points and recommended changes.
- Step-by-step journey map with user goal, system need, and risk at each step.
- Measurement plan for activation, completion, and drop-off analysis.

## Success Criteria

- Users can reach first value with minimal confusion and unnecessary work.
- The flow explains why information or permissions are needed.
- Required and optional steps are clearly distinguished.
- Recommendations are testable and feasible for product and engineering teams.

## Common Mistakes

- Adding a product tour instead of fixing unclear UI.
- Asking for detailed information before users understand value.
- Blocking progress on optional setup tasks.
- Ignoring accessibility, errors, and cancellation paths.

## Example Usage

Use this skill to review a SaaS onboarding flow that asks users to create a workspace, invite teammates, choose templates, and connect integrations.
