# Heuristic Evaluation

## Skill Name

Heuristic Evaluation

## Description

Evaluate an interface against established usability heuristics and convert observations into prioritized product and engineering recommendations.

## When To Use

- A product needs structured UX critique before user research or release.
- A team wants a fast expert review using a repeatable framework.
- A flow has usability concerns but limited analytics or research data.
- Stakeholders need a shared language for UX issues.

## Inputs Required

- Target interface, prototype, screenshots, or flow description.
- User goals, domain context, and priority tasks.
- Chosen heuristic framework or preference for Nielsen-style heuristics.
- Constraints, known issues, and success metrics.

## Step-by-Step Workflow

1. Select a heuristic framework and define the evaluated tasks.
2. Walk through the interface as a user, noting evidence for each relevant heuristic.
3. Assign severity based on frequency, impact, persistence, and workaround availability.
4. Translate each issue into actionable recommendations and validation ideas.
5. Separate heuristic concerns from open research questions.

## Output Format

- Heuristic evaluation matrix with issue, heuristic, severity, evidence, and recommendation.
- Top findings summary for stakeholders.
- Validation plan for issues that require user testing or data.

## Success Criteria

- The evaluation is systematic and not a loose opinion list.
- Severity reflects user impact and business context.
- Recommendations can be implemented or tested.
- Unknowns are framed as research questions rather than assumptions.

## Common Mistakes

- Using heuristics as labels without explaining the actual issue.
- Assigning every issue the same severity.
- Ignoring product context and user expertise level.
- Conflating visual preference with usability failure.

## Example Usage

Use this skill to evaluate a project creation wizard against visibility of status, error prevention, recognition over recall, and user control.
