# Heuristic Evaluation Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm target interface, prototype, screenshots, or flow description.
- [ ] Confirm user goals, domain context, and priority tasks.
- [ ] Confirm chosen heuristic framework or preference for Nielsen-style heuristics.
- [ ] Confirm constraints, known issues, and success metrics.

## Execution

- [ ] Select a heuristic framework and define the evaluated tasks.
- [ ] Walk through the interface as a user, noting evidence for each relevant heuristic.
- [ ] Assign severity based on frequency, impact, persistence, and workaround availability.
- [ ] Translate each issue into actionable recommendations and validation ideas.
- [ ] Separate heuristic concerns from open research questions.

## Completion

- [ ] The evaluation is systematic and not a loose opinion list.
- [ ] Severity reflects user impact and business context.
- [ ] Recommendations can be implemented or tested.
- [ ] Unknowns are framed as research questions rather than assumptions.
