# User Flow Analysis Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm flow description, screenshots, routes, prototype, analytics, or user research.
- [ ] Confirm primary user goal, entry points, success criteria, and constraints.
- [ ] Confirm known edge cases, permissions, errors, and handoffs.
- [ ] Confirm business metrics and product priorities when available.

## Execution

- [ ] Map the happy path, alternate paths, error paths, and recovery paths.
- [ ] Identify user decisions, required knowledge, system feedback, and handoffs at each step.
- [ ] Look for redundant steps, unclear copy, hidden dependencies, missing states, and accessibility risks.
- [ ] Prioritize friction by task impact, frequency, confidence, and implementation effort.
- [ ] Recommend flow changes, instrumentation, and validation checks.

## Completion

- [ ] The analysis captures real paths rather than only the ideal path.
- [ ] Recommendations reduce friction without removing necessary user control.
- [ ] Missing states and handoffs are visible to engineering.
- [ ] The team can align on the next improvement or experiment.
