# User Flow Analysis

## Skill Name

User Flow Analysis

## Description

Analyze user flows from entry point to completion, identifying decision points, friction, missing states, and opportunities for simplification.

## When To Use

- A flow spans multiple screens, roles, states, or systems.
- Users abandon midway or need support to complete a task.
- A team needs to understand the current journey before changing UI.
- A feature requires coordination between product, design, and engineering.

## Inputs Required

- Flow description, screenshots, routes, prototype, analytics, or user research.
- Primary user goal, entry points, success criteria, and constraints.
- Known edge cases, permissions, errors, and handoffs.
- Business metrics and product priorities when available.

## Step-by-Step Workflow

1. Map the happy path, alternate paths, error paths, and recovery paths.
2. Identify user decisions, required knowledge, system feedback, and handoffs at each step.
3. Look for redundant steps, unclear copy, hidden dependencies, missing states, and accessibility risks.
4. Prioritize friction by task impact, frequency, confidence, and implementation effort.
5. Recommend flow changes, instrumentation, and validation checks.

## Output Format

- User flow analysis with step map, friction points, and recommendations.
- Edge-state checklist for loading, empty, error, permission, and success states.
- Measurement and validation suggestions.

## Success Criteria

- The analysis captures real paths rather than only the ideal path.
- Recommendations reduce friction without removing necessary user control.
- Missing states and handoffs are visible to engineering.
- The team can align on the next improvement or experiment.

## Common Mistakes

- Only documenting screens without decisions or system feedback.
- Ignoring error, cancellation, and recovery paths.
- Assuming all users enter from the same place.
- Recommending fewer steps without checking comprehension and trust.

## Example Usage

Use this skill to analyze a team invitation flow from email entry through permissions, errors, confirmation, and recipient acceptance.
