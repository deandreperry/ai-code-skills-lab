# Empty State Review Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm target pages, components, or screenshots for empty, loading, error, and zero-result states.
- [ ] Confirm primary user goal and expected next action for each state.
- [ ] Confirm known data conditions, permissions, and failure modes.
- [ ] Confirm design system messaging, illustration, icon, and CTA patterns when available.

## Execution

- [ ] Inventory every non-happy-path state across the target flow.
- [ ] Clarify what the user knows, needs, and can do in each state.
- [ ] Review hierarchy, copy, CTA placement, accessibility, responsive behavior, and state transitions.
- [ ] Identify missing states such as permission-denied, filtered-empty, offline, timeout, and partial-data cases.
- [ ] Recommend concise content and UI changes that help users recover or continue.

## Completion

- [ ] Users understand why the state occurred and what they can do next.
- [ ] Empty and error states are not treated as decorative afterthoughts.
- [ ] State copy is specific, actionable, and consistent with product tone.
- [ ] Accessibility and responsive behavior are accounted for in each state.
