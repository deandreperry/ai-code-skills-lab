# Empty State Review

## Skill Name

Empty State Review

## Description

Review empty, loading, error, first-use, and zero-result states so product moments remain clear, helpful, accessible, and actionable.

## When To Use

- A screen has no data, no search results, failed loading, or first-use content.
- Users need guidance before creating their first item or recovering from an error.
- A design or implementation only covers the happy path.
- A product surface needs more polished state coverage before release.

## Inputs Required

- Target pages, components, or screenshots for empty, loading, error, and zero-result states.
- Primary user goal and expected next action for each state.
- Known data conditions, permissions, and failure modes.
- Design system messaging, illustration, icon, and CTA patterns when available.

## Step-by-Step Workflow

1. Inventory every non-happy-path state across the target flow.
2. Clarify what the user knows, needs, and can do in each state.
3. Review hierarchy, copy, CTA placement, accessibility, responsive behavior, and state transitions.
4. Identify missing states such as permission-denied, filtered-empty, offline, timeout, and partial-data cases.
5. Recommend concise content and UI changes that help users recover or continue.

## Output Format

- State coverage matrix with current and recommended behavior.
- Prioritized findings for missing, confusing, or inaccessible states.
- Suggested copy, CTA, and validation notes for each important state.

## Success Criteria

- Users understand why the state occurred and what they can do next.
- Empty and error states are not treated as decorative afterthoughts.
- State copy is specific, actionable, and consistent with product tone.
- Accessibility and responsive behavior are accounted for in each state.

## Common Mistakes

- Using the same generic message for every empty state.
- Offering a CTA that users cannot actually complete.
- Forgetting filtered, permission, and network-related empty states.
- Relying on illustration without useful text or action.

## Example Usage

Use this skill to review a dashboard with no projects, failed analytics loading, zero search results, and permission-limited views.
