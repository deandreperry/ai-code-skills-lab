# UX Audit

## Skill Name

UX Audit

## Description

Audit product experience for clarity, task completion, interaction quality, error recovery, content, accessibility, and visual hierarchy.

## When To Use

- A product surface feels confusing, slow, or inconsistent.
- A team needs prioritized UX issues before redesigning a flow.
- Analytics or support data suggests users are getting stuck.
- A feature needs senior UX review before launch.

## Inputs Required

- Target flow, screenshots, prototype, URL, or product description.
- Primary users, goals, constraints, and success metrics.
- Known pain points, analytics, support tickets, or usability feedback.
- Design system and accessibility requirements.

## Step-by-Step Workflow

1. Define the primary user goal, entry points, completion criteria, and constraints.
2. Review the flow for clarity, hierarchy, affordance, feedback, errors, accessibility, responsiveness, and content quality.
3. Distinguish usability blockers from polish issues and design preferences.
4. Prioritize findings by impact on task completion and confidence in evidence.
5. Recommend incremental improvements with product and engineering implications.

## Output Format

- UX audit report with severity, evidence, and recommendations.
- Prioritized issue list mapped to user goals.
- Suggested experiments, instrumentation, or research follow-up when evidence is incomplete.

## Success Criteria

- Findings explain how users are affected, not just what looks imperfect.
- Recommendations are specific and feasible within product constraints.
- Accessibility and content quality are included in UX judgment.
- The team can choose a first improvement without a redesign mandate.

## Common Mistakes

- Treating personal taste as user impact.
- Auditing screens without understanding the user goal.
- Ignoring error, empty, loading, and edge states.
- Suggesting broad redesigns when targeted fixes would work.

## Example Usage

Use this skill to audit a dashboard onboarding flow for clarity, empty states, call-to-action hierarchy, and error recovery.
