# Information Architecture Audit Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm sitemap, navigation structure, routes, content inventory, or screenshots.
- [ ] Confirm user segments, top tasks, terminology, and analytics when available.
- [ ] Confirm known findability problems, support tickets, or search logs.
- [ ] Confirm technical constraints for routing, permissions, or content management.

## Execution

- [ ] Inventory current navigation, labels, route hierarchy, content groups, and entry points.
- [ ] Map structure against user tasks, mental models, permissions, and product domains.
- [ ] Identify duplicate labels, ambiguous categories, orphaned content, and over-deep hierarchies.
- [ ] Recommend labeling, grouping, navigation, breadcrumb, and search improvements.
- [ ] Define validation methods such as tree testing, analytics review, or usability tasks.

## Completion

- [ ] Users can predict where important content or features belong.
- [ ] Labels use audience language rather than internal org structure.
- [ ] The structure can scale as new content is added.
- [ ] Recommendations respect permissions, routing, and technical constraints.
