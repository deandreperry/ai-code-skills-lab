# Information Architecture Audit

## Skill Name

Information Architecture Audit

## Description

Audit navigation, labeling, grouping, hierarchy, findability, and content organization for complex product or documentation experiences.

## When To Use

- Users struggle to find features, settings, docs, or support content.
- Navigation has grown organically and needs rationalization.
- A product or docs site is adding new sections and needs scalable structure.
- Search, menus, breadcrumbs, or page hierarchy feel inconsistent.

## Inputs Required

- Sitemap, navigation structure, routes, content inventory, or screenshots.
- User segments, top tasks, terminology, and analytics when available.
- Known findability problems, support tickets, or search logs.
- Technical constraints for routing, permissions, or content management.

## Step-by-Step Workflow

1. Inventory current navigation, labels, route hierarchy, content groups, and entry points.
2. Map structure against user tasks, mental models, permissions, and product domains.
3. Identify duplicate labels, ambiguous categories, orphaned content, and over-deep hierarchies.
4. Recommend labeling, grouping, navigation, breadcrumb, and search improvements.
5. Define validation methods such as tree testing, analytics review, or usability tasks.

## Output Format

- Information architecture audit with findings and proposed structure.
- Labeling and navigation recommendations with rationale.
- Validation plan and migration considerations.

## Success Criteria

- Users can predict where important content or features belong.
- Labels use audience language rather than internal org structure.
- The structure can scale as new content is added.
- Recommendations respect permissions, routing, and technical constraints.

## Common Mistakes

- Reorganizing around internal teams instead of user tasks.
- Creating too many top-level categories.
- Ignoring search logs and support signals when available.
- Renaming labels without considering migration and analytics impact.

## Example Usage

Use this skill to audit a developer docs navigation system and propose clearer groupings for guides, API references, examples, and migration notes.
