# Codebase Onboarding Guide Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm repository tree, package scripts, environment variables, and setup requirements.
- [ ] Confirm framework, deployment target, testing stack, and major dependencies.
- [ ] Confirm domain overview and key user journeys when available.
- [ ] Confirm team conventions for branches, commits, reviews, and releases.

## Execution

- [ ] Inspect package metadata, docs, app entry points, routing, feature folders, and test setup.
- [ ] Identify the fastest path to install, run, test, lint, and build the project.
- [ ] Explain the architecture in terms of practical contribution paths rather than abstract theory.
- [ ] Document common tasks such as adding a route, component, test, API call, or design token.
- [ ] Call out risks, missing documentation, and the best first issues for a new contributor.

## Completion

- [ ] A new contributor can get the app running without tribal knowledge.
- [ ] The guide reflects the actual repository rather than idealized conventions.
- [ ] Risky areas and ownership boundaries are visible.
- [ ] The document is concise enough to be maintained.
