# Responsive Design Audit Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm target pages, components, or flows.
- [ ] Confirm supported viewport range, browser matrix, and zoom expectations.
- [ ] Confirm known content edge cases such as long names, empty states, tables, or dense controls.
- [ ] Confirm screenshots, design references, or bug reports when available.

## Execution

- [ ] Identify the layout model, critical breakpoints, container constraints, and fixed-format UI elements.
- [ ] Check navigation, forms, tables, media, dialogs, and repeated cards at mobile, tablet, desktop, and zoomed states.
- [ ] Test long content, empty content, reduced width, orientation changes, and touch targets.
- [ ] Recommend CSS changes that use fluid constraints, container queries, responsive grids, and stable dimensions where appropriate.
- [ ] Summarize failures with reproduction steps and priority based on user impact.

## Completion

- [ ] No critical workflow is blocked at supported viewport sizes.
- [ ] Text remains readable and contained without overlap or clipping.
- [ ] Interactive targets are reachable by touch, pointer, and keyboard.
- [ ] Layout changes are stable and consistent with the existing design system.
