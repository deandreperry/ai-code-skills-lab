# Color Contrast Token Audit Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm color token files, theme files, design specs, screenshots, or component examples.
- [ ] Confirm target WCAG contrast expectations and supported themes.
- [ ] Confirm text, icon, border, status, focus, disabled, and chart color usage.
- [ ] Confirm known accessibility bugs or brand constraints.

## Execution

- [ ] Inventory foreground/background token pairings across semantic roles and states.
- [ ] Check text, icons, focus indicators, status messaging, borders, and disabled treatments.
- [ ] Evaluate light, dark, brand, hover, active, selected, error, warning, and success states.
- [ ] Identify token pairings that fail or are too fragile for real content.
- [ ] Recommend token changes, semantic pairing rules, and component-level safeguards.

## Completion

- [ ] Critical text and controls meet contrast expectations across supported themes.
- [ ] Designers and developers know which token pairings are safe.
- [ ] Focus, error, and status states remain perceivable.
- [ ] Recommendations preserve brand intent where possible.
