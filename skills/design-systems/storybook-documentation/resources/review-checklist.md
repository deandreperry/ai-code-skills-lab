# Storybook Documentation Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm component source, existing stories, docs pages, design specs, and usage examples.
- [ ] Confirm supported Storybook version and add-ons.
- [ ] Confirm variant matrix, states, accessibility requirements, and known edge cases.
- [ ] Confirm testing goals such as interaction tests or visual snapshots.

## Execution

- [ ] Inventory the component API, variants, states, slots, and common usage patterns.
- [ ] Create stories that demonstrate real product usage rather than decorative demos.
- [ ] Add controls, docs, accessibility notes, interaction examples, and edge states.
- [ ] Keep story names and organization consistent with the design system taxonomy.
- [ ] Verify stories render, avoid brittle mocks, and support regression testing when possible.

## Completion

- [ ] Consumers can understand when and how to use the component.
- [ ] Stories cover default, variant, state, error, loading, disabled, and edge content cases as relevant.
- [ ] Accessibility expectations are visible and testable.
- [ ] Stories remain maintainable as the component evolves.
