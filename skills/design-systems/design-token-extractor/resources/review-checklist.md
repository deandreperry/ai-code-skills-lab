# Design Token Extractor Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm style files, theme files, component library, or representative screenshots.
- [ ] Confirm existing token format or desired output target such as CSS variables, JSON, or TypeScript.
- [ ] Confirm brand constraints, naming conventions, and platform requirements.
- [ ] Confirm priority surfaces or components to analyze first.

## Execution

- [ ] Inventory repeated visual values and group them by semantic role, not just raw value.
- [ ] Separate primitive tokens from semantic tokens and component-specific aliases.
- [ ] Detect near-duplicates that may need consolidation or design review.
- [ ] Recommend naming, scale structure, and migration strategy that match the current stack.
- [ ] Produce token output and a safe adoption plan for the highest-value components.

## Completion

- [ ] Tokens reflect product meaning and are not only color or spacing constants.
- [ ] Naming is consistent, discoverable, and extensible.
- [ ] Existing visual output can be preserved during migration.
- [ ] The result supports future theming and design system governance.
