# Design Handoff Review Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm figma frames, screenshots, specs, prototype links, or design export notes.
- [ ] Confirm target platform, component library, and design system constraints.
- [ ] Confirm user flows, acceptance criteria, and supported breakpoints.
- [ ] Confirm known engineering constraints, data dependencies, and edge cases.

## Execution

- [ ] Map the intended user flow and implementation surface from the design materials.
- [ ] Check states, variants, responsive rules, content behavior, tokens, assets, and interaction details.
- [ ] Review accessibility requirements including semantics, focus, contrast, labels, keyboard, and motion.
- [ ] Identify ambiguity that could cause inconsistent implementation or rework.
- [ ] Return a handoff readiness assessment with questions and required additions.

## Completion

- [ ] Engineers can implement without inventing important product or design behavior.
- [ ] Designers receive specific, actionable gaps to resolve.
- [ ] Accessibility and responsive requirements are visible before coding.
- [ ] Handoff questions are prioritized by implementation risk.
