# Typography System Review

## Skill Name

Typography System Review

## Description

Audit typography scales, semantic text styles, readability, hierarchy, line length, responsive behavior, and token usage across a product or design system.

## When To Use

- A product feels visually inconsistent or hard to scan because of type usage.
- A design system is defining or revising typography tokens.
- Components use ad hoc font sizes, line heights, weights, or letter spacing.
- A team needs typography guidance that works across design and code.

## Inputs Required

- Typography tokens, CSS, design specs, screenshots, or component examples.
- Brand typefaces, supported platforms, localization, and accessibility constraints.
- Content types such as headings, body text, labels, captions, tables, and code.
- Responsive and density requirements.

## Step-by-Step Workflow

1. Inventory type styles, tokens, and real usage across representative surfaces.
2. Review hierarchy, readability, line height, line length, weight, spacing, and responsive behavior.
3. Separate primitive font tokens from semantic text roles and component-specific use.
4. Identify inconsistencies, inaccessible sizing, cramped layouts, and localization risks.
5. Recommend a scalable typography system and migration plan.

## Output Format

- Typography audit with findings and recommended token/style changes.
- Semantic type style inventory for design and engineering.
- Migration and QA checklist for applying typography updates safely.

## Success Criteria

- Text hierarchy is clear and consistent across product surfaces.
- Typography supports readability, accessibility, localization, and responsive layouts.
- Tokens express semantic roles instead of one-off values.
- Recommendations are practical for both designers and developers.

## Common Mistakes

- Choosing type scale values without checking real content.
- Using viewport-based font sizing that harms layout stability.
- Treating labels, helper text, and table text as afterthoughts.
- Ignoring localization, zoom, and dynamic content length.

## Example Usage

Use this skill to audit typography tokens and component usage before a design system refresh.
