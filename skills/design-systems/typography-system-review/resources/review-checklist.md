# Typography System Review Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm typography tokens, CSS, design specs, screenshots, or component examples.
- [ ] Confirm brand typefaces, supported platforms, localization, and accessibility constraints.
- [ ] Confirm content types such as headings, body text, labels, captions, tables, and code.
- [ ] Confirm responsive and density requirements.

## Execution

- [ ] Inventory type styles, tokens, and real usage across representative surfaces.
- [ ] Review hierarchy, readability, line height, line length, weight, spacing, and responsive behavior.
- [ ] Separate primitive font tokens from semantic text roles and component-specific use.
- [ ] Identify inconsistencies, inaccessible sizing, cramped layouts, and localization risks.
- [ ] Recommend a scalable typography system and migration plan.

## Completion

- [ ] Text hierarchy is clear and consistent across product surfaces.
- [ ] Typography supports readability, accessibility, localization, and responsive layouts.
- [ ] Tokens express semantic roles instead of one-off values.
- [ ] Recommendations are practical for both designers and developers.
