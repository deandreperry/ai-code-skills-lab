# Developer Experience Audit Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm repository structure, README, docs, package scripts, setup files, and CI workflow.
- [ ] Confirm target contributor profile and expected first tasks.
- [ ] Confirm known setup failures, support questions, or onboarding feedback.
- [ ] Confirm required local tools, secrets, fixtures, and verification commands.

## Execution

- [ ] Walk the repository as a new contributor from clone to first verified change.
- [ ] Review setup, scripts, environment variables, docs, examples, tests, linting, and CI alignment.
- [ ] Identify confusing commands, missing prerequisites, slow feedback loops, and unclear errors.
- [ ] Evaluate whether docs explain contribution paths and architectural conventions.
- [ ] Recommend quick wins and structural DX improvements with priority and effort.

## Completion

- [ ] A contributor can understand and verify the project with minimal outside help.
- [ ] Commands and docs match the actual repository behavior.
- [ ] Pain points are prioritized by onboarding and maintenance impact.
- [ ] Recommendations improve speed and confidence without unnecessary tooling.
