# README Upgrader Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm existing README, package metadata, docs, scripts, and repository structure.
- [ ] Confirm project audience, purpose, and maturity level.
- [ ] Confirm setup requirements, environment variables, and verification commands.
- [ ] Confirm contribution and release expectations.

## Execution

- [ ] Inspect the repository before writing marketing-style copy.
- [ ] Create an information hierarchy: overview, value, architecture, setup, usage, verification, contribution, roadmap, and support.
- [ ] Preserve accurate existing information while removing outdated or redundant content.
- [ ] Add command examples and links to deeper docs where useful.
- [ ] Keep tone professional, specific, and honest about current capabilities.

## Completion

- [ ] A reader can understand what the project is and how to use it within minutes.
- [ ] Commands and paths reflect the actual repository.
- [ ] The README supports both quick evaluation and deeper contribution.
- [ ] Claims are accurate and not inflated.
