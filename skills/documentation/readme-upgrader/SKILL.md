# README Upgrader

## Skill Name

README Upgrader

## Description

Upgrade a README into a clear, professional entry point that explains purpose, setup, architecture, usage, testing, and contribution flow.

## When To Use

- A repository README is incomplete, stale, or too shallow for serious contributors.
- A project needs to look credible to hiring teams, customers, or open source users.
- Developers cannot quickly understand how to run, test, or extend the project.
- Documentation should be improved without exaggerating project maturity.

## Inputs Required

- Existing README, package metadata, docs, scripts, and repository structure.
- Project audience, purpose, and maturity level.
- Setup requirements, environment variables, and verification commands.
- Contribution and release expectations.

## Step-by-Step Workflow

1. Inspect the repository before writing marketing-style copy.
2. Create an information hierarchy: overview, value, architecture, setup, usage, verification, contribution, roadmap, and support.
3. Preserve accurate existing information while removing outdated or redundant content.
4. Add command examples and links to deeper docs where useful.
5. Keep tone professional, specific, and honest about current capabilities.

## Output Format

- Updated README or structured rewrite proposal.
- Summary of major documentation improvements.
- Follow-up documentation gaps when outside README scope.

## Success Criteria

- A reader can understand what the project is and how to use it within minutes.
- Commands and paths reflect the actual repository.
- The README supports both quick evaluation and deeper contribution.
- Claims are accurate and not inflated.

## Common Mistakes

- Writing a landing page instead of usable project documentation.
- Inventing features or install steps that do not exist.
- Overloading the README with details better suited for docs pages.
- Forgetting validation, testing, and contribution commands.

## Example Usage

Use this skill to turn a barebones component library README into a senior-level project overview with setup, architecture, examples, and roadmap.
