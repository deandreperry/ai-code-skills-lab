# Technical Doc Generator Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm code paths, feature description, diagrams, API contracts, and related tickets.
- [ ] Confirm target audience and expected depth.
- [ ] Confirm existing documentation style and location.
- [ ] Confirm known caveats, operational steps, and verification commands.

## Execution

- [ ] Inspect the relevant code, tests, configuration, and existing docs before drafting.
- [ ] Define the audience, document purpose, and scope boundaries.
- [ ] Explain concepts, architecture, data flow, usage, configuration, testing, and troubleshooting as appropriate.
- [ ] Use examples and command snippets that match the repository.
- [ ] Call out limitations, ownership, and update triggers so the doc stays useful.

## Completion

- [ ] The document helps the target audience complete real tasks.
- [ ] Examples are accurate and grounded in current code.
- [ ] The level of detail matches the audience.
- [ ] The doc has a clear owner, location, and maintenance signal.
