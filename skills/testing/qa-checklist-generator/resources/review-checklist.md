# QA Checklist Generator Review Checklist

Use this checklist while executing the skill.

## Preparation

- [ ] Confirm feature description, acceptance criteria, designs, and user flows.
- [ ] Confirm affected roles, permissions, devices, browsers, and locales.
- [ ] Confirm known risks, dependencies, and existing automated coverage.
- [ ] Confirm release timeline and environments available for testing.

## Execution

- [ ] Identify core user tasks, permissions, state transitions, and failure modes.
- [ ] Group checks by workflow, state, platform, accessibility, analytics, and regression risk as relevant.
- [ ] Make each checklist item observable and pass/fail oriented.
- [ ] Separate smoke checks, full regression checks, and exploratory prompts.
- [ ] Include setup data and environment notes so the checklist can be executed reliably.

## Completion

- [ ] The checklist is short enough to use but broad enough to catch likely issues.
- [ ] Each item has a clear expected result.
- [ ] Critical accessibility, responsive, and error states are included.
- [ ] Manual effort is focused where automation is weak or unavailable.
